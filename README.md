# WhitelistChecker (iOS · macOS)

Диагностика мобильной сети в режиме «белых списков»: для каждого адреса определяет,
**заблокирован** он, **зашейплен** или идёт на **полной скорости** (в белом списке).

Один и тот же код собирается под **iOS/iPadOS** и под **macOS (Apple Silicon)** —
десктопная версия через Mac Catalyst, без отдельной ветки кода.

## Что делает

Два режима:

### Шейп
Список доменов вводить **не нужно**. Меряется скорость канала по встроенным
эталонам: белый whitelisted сервер (зеркало Яндекса) против чужих CDN
(Cloudflare, Google, GitHub). Если чужой трафик заметно медленнее белого —
это шейп по белым спискам. Панель «Что реально измерено» показывает фактическую
скорость каждого эталона и итоговое соотношение.

> Per-site замер скорости намеренно убран: большинство сайтов отдаёт по `/`
> мелкое тело и канал не насыщает, поэтому надёжный сигнал — только сравнение
> эталонов на уровне сети.

### Блокировка
Ввод — **IP, CIDR-подсеть или домен** (по одному в строке). Проверка многоуровневая:

1. **TCP-handshake к :443** (`NWConnection` tcp): `OPEN` / `RST` / `DROP`.
2. **TLS/HTTPS-проба** (для доменов с открытым TCP): проходит ли TLS-рукопожатие с
   нужным SNI. Ловит блокировку по **SNI/DPI**, когда TCP открывается, но рукопожатие
   рвут — голая TCP-проба такое пропускает. После рукопожатия добирается HTTP-статус.
3. **ICMP-пинг** (`SOCK_DGRAM`, без entitlement) — **справочно**, на вердикт не влияет:
   многие хосты и CDN глушат ICMP по политике, оставаясь доступными.

Вердикты:
- 🟢 **ДОСТУПЕН** — TCP открыт и TLS-рукопожатие прошло (для IP — только TCP)
- ⛔ **BLOCKED** — TCP не устанавливается (`RST`/`DROP`) **или** TLS рвут по SNI/DPI
- ⚪ **INCONCLUSIVE** — DNS не резолвится / нет данных

CIDR разворачивается в TCP-карту по всем адресам подсети (авторазворот до `/24`;
крупнее — запрос подтверждения).

Поверх обоих режимов — **агрегатный «режим сети»**: блокировка по белым спискам,
шейп по белым спискам или ограничений не видно.

## Импорт списка
В режиме «Блокировка» список целей можно загрузить тремя способами — перед
заливкой в поле ввода он **валидируется** (сколько строк распознано с разбивкой
IP / CIDR / домены и какие пропущены с причиной), затем **Заменить ввод** или
**Добавить к вводу**. Формат: по одной цели в строке, `#` — комментарий. Большие
CIDR на этом этапе не разворачиваются (разворот — при запуске проверки).

1. **Вставить** — из буфера обмена. Работает при любой подписи.
2. **Импорт .txt** — системный пикер файлов.
3. **«Открыть в WL Checker»** — из Файлов/почты через системный Share-лист
   (приложение зарегистрировано как обработчик `.txt`/`.csv`).

> ⚠️ **Подпись и импорт файлов.** Пункты 2–3 используют системный Document
> Picker, который ходит вне процесса приложения и завязан на entitlements из
> provisioning profile. При переподписи **сторонним покупным сертификатом**
> (напр. Gbox) entitlements могут не совпасть — тогда выбор файла молча не
> срабатывает. При подписи **своим** сертификатом (SideStore/Sideloadly) всё
> работает. Универсальный обход — **Вставить** из буфера (не зависит от подписи).

## Экспорт лога
Кнопка **«Поделиться»** (↑ в правом верхнем углу) собирает подробный текстовый
отчёт о последней проверке и отдаёт его в системный share-лист iOS (сохранить в
Файлы, отправить в мессенджер, AirDrop и т.д.). В отчёте: версия/билд, дата,
режим, статус DNS, режим сети, замеры эталонов с итогом по шейпу и построчные
результаты целей (вердикт, IP, TCP, скорость, детали).

## DNS
Системный DNS телефона приложение менять не может, но для своих проб может
резолвить домены через выбранный сервер (Yandex `77.88.8.8` / оператора /
системный). Если системный DNS не отвечает — кнопки переключения резолвера
появляются автоматически.

## Сборка (без подписи)

**iOS:**
```bash
./build-ipa.sh
# → dist/WhitelistChecker-unsigned.ipa
```
На выходе — **голый unsigned .ipa**. Подпись — отдельно: Sideloadly / Feather / Gbox
своим сертификатом разработчика.

**macOS (Apple Silicon):**
```bash
./build-dmg.sh
# → dist/WhitelistChecker.dmg   (Mac Catalyst, arm64)
```
Образ **оформленный**: крупные иконки, фикс. размер окна, приложение слева — ярлык
`Applications` справа. Раскладку пишет [`dmgbuild`](https://pypi.org/project/dmgbuild/)
(pure-Python, правит `.DS_Store` напрямую — без Finder/AppleScript, поэтому стабильно
работает и в headless CI). Скрипт сам ставит его в локальный venv `.dmgvenv` (нужны
`python3` и сеть); если установка не удалась — собирается простой `.dmg`, сборка не падает.

В образе — `WhitelistChecker.app` и ярлык `Applications`: перетащить — установить.
Приложение подписано **ad-hoc** (без Apple ID), поэтому при скачивании из сети
macOS поставит карантин. Снять перед первым запуском:
```bash
xattr -dr com.apple.quarantine /Applications/WhitelistChecker.app
```
(или ПКМ по приложению → «Открыть»). Минимальная версия — macOS 13.

Требуется Xcode 16+ (проект использует synchronized folder groups, objectVersion 77).

## Автосборка через GitHub Actions

К репозиторию подключён CI (`.github/workflows/build.yml`): на стороне GitHub
параллельно собираются **оба** артефакта — iOS `.ipa` и macOS `.dmg` — локальный
Xcode не нужен. Сборка запускается **на каждый коммит в `main`**.

**Где взять готовые сборки:**
- **Nightly (свежий билд):** релиз **`nightly`** в разделе Releases — скользящий
  prerelease, который после каждого коммита в `main` пересобирается и указывает на
  последнюю версию. В нём лежат и `.ipa`, и `.dmg`.
- **Стабильный релиз:** push тега `vX.Y.Z` (`git tag v1.2 && git push github v1.2`)
  → отдельный **GitHub Release** этого тега с обоими файлами.
- **Вручную:** вкладка **Actions → build → Run workflow**; артефакты `ipa` и `dmg`
  лежат в **Artifacts** прогона.

`.ipa` подпиши своим сертификатом (Sideloadly / Feather / Gbox) и поставь на
телефон. `.dmg` — поставь на Mac (Apple Silicon), сняв карантин (см. выше).
Полноценная подпись/нотаризация в CI не делается — для неё нужен личный
Apple-сертификат, который в репозиторий класть нельзя.

## Ограничения
- Per-site скорость произвольного сайта надёжно не снять (мелкое тело по `/`),
  поэтому шейп определяется по сравнению встроенных эталонов, а не по введённым
  доменам.
- Эталоны качаются параллельно и делят канал, поэтому их абсолютные скорости ниже
  реального потолка каждого — но соотношение белый/чужой (сам индикатор шейпа)
  остаётся валидным.
- raw ICMP/SYN на iOS недоступны без спец-entitlement, но **непривилегированный
  ICMP** (`socket(AF_INET, SOCK_DGRAM, IPPROTO_ICMP)`, как в Apple SimplePing) работает
  без прав — используется для справочного пинга. Block-вердикт строится на
  `NWConnection` (TCP + TLS/SNI), ICMP на него не влияет.

## Структура
```
WhitelistChecker/
├─ Engine/
│  ├─ InputParser.swift   — разбор/валидация IP/CIDR/домена, разворот подсети, импорт
│  ├─ Types.swift         — CheckMode, TCPResult, Verdict, ProbeResult, NetworkMode
│  ├─ DNSResolver.swift   — резолвинг через выбранный сервер (getaddrinfo / UDP-53)
│  ├─ TCPProbe.swift      — block-сигнал (NWConnection tcp): OPEN/RST/DROP
│  ├─ HTTPSProbe.swift    — TLS/SNI-проба поверх TCP (ловит DPI-сброс), HTTP-статус
│  ├─ ICMPPing.c/.h       — справочный ICMP-пинг (непривилегированный SOCK_DGRAM)
│  ├─ ThroughputProbe.swift — замер скорости (NWConnection tls + HTTP GET + Range)
│  ├─ Calibration.swift   — встроенные эталоны, замер белый/чужой, вердикт по сети
│  └─ ScanEngine.swift    — оркестратор, классификация, режим сети
├─ UI/
│  ├─ ContentView.swift   — ввод, импорт .txt, экспорт лога, DNS, баннер, эталоны, список
│  └─ RowView.swift       — строка результата
└─ WhitelistCheckerApp.swift

assets/dmg/
└─ dmg-settings.py        — раскладка окна .dmg для dmgbuild (позиции, размер, иконки)
```
